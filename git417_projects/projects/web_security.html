Project 4: Web Security

For this week's project, I created a page outlining web security from the perspective of a site visitor, a developer or website manager, and a JavaScript student. Users can read the web safety summaries, click on as many links as they wish without spamming their browser with duplicate tabs, and see a little bit of what information JavaScript can access from their computer by opening the widgets.

Web Safety Basics

Although it's difficult to navigate the web safely and privately, there are a couple of ways that will help you have a private, safer web surfing experience.

For the more secure browsing, security editor Zack Whittaker proposes a combination of using a VPN or creating your own VNP server, using a secure DNS such as Cloudflare, visiting mostly HTTPS-enabled sites, replacing insecure plug-ins for ad- and tracker-blocking extensions, and using Tor through a Tor browser.

Webroot, a company that protects businesses and consumers from cyber threats, advises that any link can lead you into spyware traps that steal your data without your consent, that sites such as social networking, adult, and torrenting sites are common phishing sites, and that you should always keep your computer, browsers, and software updated with the newest and most secure settings. Unsurprisingly, they also suggest getting an antivirus program like the one which they provide.

Editor of TheBestVPN.com, Rob Mardisalu, suggests several tips for safe Internet browsing. Of the tips not mentioned above, new ones include downloading the most secure browser you can find, using a password manager with unique passwords instead of saving simple passwords on browser autofill options, learning how to spot and avoid phishing emails, downloading software from and shopping on trustworthy sites, using two-factor authentication, changing passwords after a data breach, using credit monitoring, only hopping on secure wifi networks, and backing up your data.

On the other hand, for developers and others who are trying to secure their websites for themselves and site visitors, Lars Lofgren from Quick Sprout provides similar tips. However, some new tips include using an HTTPS portocol with an SSL certificate, choosing a safe web hosting provider, using tools to monitor your site's security, changing default CMS settings, and restricting file uploads from site visitors.

And lastly, the best weapon in your arsenal is your common sense. If something seems risky, trust your intuition and avoid it.

Finding Public Site Visitor Information

A lot of information, such as a user's operating system and screen resolution, is easily accessible through JavaScript without the need for user input. To view information about your own browser and computer, and to learn how to find that information through code in your own website, view the sections below.

(None of the methods used to collect this shared information poses a security risk to the site visitor, nor to their device.)

Web Browser Name

Your current web browser is shown as [Browser] by using the navigator object's appName property.

Unless you're using Internet Explorer 10 (or an earlier version) or Opera, your browser will most likely return "Netscape" as the answer, which is why this property alone should no longer be relied upon for finding a user's exact browser.

Operating System

Your current operating system is shown as [User Platform] by using the navigator object's platform property.

Newer browsers such as Firefox and Chrome will return Win32 even on 64-bit operating systems, which is why this method of learning a user's operating system is no longer as reliable as it used to be.

Screen Resolution

Your current screen resolution is shown as [Insert Screen Resolution Here] by using the screen object's height and width properties.

Browser Window Size

The dimensions of your browser window when the page loaded is shown as approximately [Browser Window Size] by using the window object's outerHeight and outerWidth properties.

Available Size for Content

The available size for web content on your screen is shown as [Available Content Size] by using the screen object's availHeight and availWidth properties.

If you are using an operating system such as a Mac, you will notice a reasonable difference between the available height as shown here and the full height of your screen resolution! This is because the Dock at the bottom would be subtracted from the full height.

Color Depth

Your current screen's color depth is shown as [User Platform] by using the screen object's colorDepth property.

Did you know that 24-bit color depth screens are now the standard for phones and computers?
