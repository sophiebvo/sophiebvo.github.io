Project 3: Exception Handling

For this week's project, I created a contact form and experimented with exception handling to show the user different messages if the input fields are not properly filled out when attempting to submit the form. Although the form does not actually send the submitted data anywhere, it responds in a way that makes it user friendly to understand if the form has or has not been correctly submitted. The entire form disappears after a successful submission to prevent the user from accidentally resubmitting the form, and to put emphasis on a thank you message which implies to the user that the form has been properly submitted.
