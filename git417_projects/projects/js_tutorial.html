Project 6: JavaScript Tutorial

Changing Content With CSS Through JavaScript

Although HTML and CSS alone can be combined to create beautiful and functional websites, JavaScript can help you add more complex features to your content, and add new content altogether. This basic tutorial which I have created will attempt to briefly teach you how to access content displayed in your web page, how to change that content, and how to change CSS through JavaScript to change the way your content is displayed. (Any links below refer to W3Schools and MND Web Docs pages which are provided for additional information.)

Accessing an Element

In order to change the way something is displayed on your web page, you first need to be able to access what is being displayed on your web page. JavaScript can interact with the Document Object Model (DOM) of your page to do this, with all of the elements on your web page treated as individual objects. You can access an HTML document's element, content, properties, and attributes using various JavaScript methods, some of which we will go over below. This is how you will be able to then change the way the content is displayed in your web page.

You can access elements in the DOM by using the document object and its methods. One of the easiest methods to use is the getElementByID() method, which is the method we will use for all the interactive coding examples in this tutorial. By giving an HTML tag an id attribute, you can easily target that specific element in your HTML document when writing JavaScript code to interact with it.

For example, if you wanted to interact with just one paragraph somewhere in your document, this is how you could go about targeting that paragraph:

HTML Code Snippet:

                        
<p id="normalPar">Lorem ipsum dolor sit amet...</p>

JS Code Snippet:

                        
var myParagraph = document.getElementById("normalPar");

Another useful method is the getElementsByClassName() method, which lets you access all elements with a particular class name. For example, if you wanted to target all paragraphs in your document that you had styled with red font, this is how you might go about doing it:

HTML Code Snippet:

                        
<p class="redFont">Lorem ipsum dolor sit amet...</p>
<p class="redFont">Viverra vitae congue eu consequat...</p>

CSS Code Snippet:

                        
.redfont {
    color: red;
}

JS Code Snippet:

                        
var redParagraphs = document.getElementsByClassName("redFont");

You can also target collections of elements by their tag name (getElementsByTagName()), by their CSS selectors (querySelectorAll()), and (for forms) by their name (getElementsByName()), which can all be read about here. For these three methods and the getElementsByClassName() method, the list of elements is indexed when you target them, so using array syntax, you can target just one of these elements if you wish. Because index numbers begin with 0 and go up from there, this is how you would alter the JavaScript code in the last example to target just the second paragraph instead of all of all of the paragraphs:

JS Code Snippet:

                        
var redParagraphs = document.getElementsByClassName("redFont")[1];

For this tutorial, you will only need to know how to use the getElementByID() method, but I highly recommend trying to recreate this tutorial with different methods on your own time to better understand the pros and cons of using one or the other.

Accessing and Changing an Element's Content

In order to change the text of an element in your document, you will need to use the textContent property. Although we can also use the innerHTML or innerText properties for this, because innerHTML is generally unsafe due to its cross-site scripting vulnerabilities, we will avoid that one. innerText could be used, but for the purposes of the interactive examples in this tutorial, textContent has slightly better compatibility with older browsers and does what it needs to. Read more here if you're curious about the way the three properties behave differently.

If you want to change the text of a heading in your document after the click of a button, one way to achieve this is by simply accessing the header element using the getElementByID() method, changing its text using the innerText property, accessing the button element by the same getElementByID() method, and adding an event listener so that the function triggers when you click on a button:

HTML Code Snippet:

                        
<h2 id="changingH2">Clicking on buttons is fun!</h2>
<button type="button" id="button1">Click Me</button>

JS Code Snippet:

                        
function changeText() {
    var elementText = document.getElementById("changingH2"); //accesses header
    elementText.textContent = "Thanks for clicking on the button!"; //changes text
}
                            
var btn1 = document.getElementById("button1"); //accesses button
btn1.addEventListener("click", changeText, false); //clicking triggers function

Preview:

We've now managed to change the text, but clicking the button a second time doesn't appear to do anything since there are no conditional statements to make something different happen once the button has been clicked more than once. One way to add a bit more interactivity would be by toggling between messages.

If you want to toggle between one text state and the other whenever you click on the button, this can be achieved by slightly changing the JavaScript code to add simple if and else statements and some variables to store strings. Using the same HTML code snippet as above, here is one way that this could be achieved by altering the last JavaScript code snippet:

JS Code Snippet:

                        
function changeText() {
    var elementText = document.getElementById("changingH2");
    var firstState = "Clicking on buttons is fun!"; //1st and original state
    var secondState = "Click on it again!"; //2nd state
                    
    if (elementText.textContent === firstState) { //if text matches 1st state,
        elementText.textContent = secondState; //change to 2nd state text
    } else { //else (if text does not match 1st state),
        elementText.textContent = firstState; //change to 1st state text
    }
}

var btn1 = document.getElementById("button1");
btn1.addEventListener("click", changeText, false);

Assessing and Changing an Element's CSS Properties

Now we have a bit more interactivity by changing the element's content, but the text still looks a bit dull. Luckily, JavaScript can access CSS properties through the DOM. This means that elements existing in the DOM can be changed using JavaScript functions, and that new elements created in JavaScript and added to the DOM tree can have CSS styling applied to them as well. This is extremely useful when you have conditional statements where you want certain actions to occur in certain circumstances.

Applying the style property followed by the CSS property's name in dot notation references an element's CSS properties. The property's name will always be written the same unless the CSS property has a hyphen in it, in which case the hyphen will be removed and the following letter will be capitalized.

For example, if you want to change the default background color, padding, border radius, text alignment, and font color of the element in the last example when the text is "Click on it again!" but want to have them returned to the way it was displayed before when the text returns to "Clicking on buttons is fun!" then you would use the style property followed by the appropriate CSS property name and set it to the values you wish. To remove a style you just added and return it to the way it looked before, set its value to an empty string. Keeping the HTML code the same, here is one way that you could alter the last JavaScript code snippet to achieve this:

JS Code Snippet:

                        
function changeText() {
    var elementText = document.getElementById("changingH2");
    var firstState = "Clicking on buttons is fun!";
    var secondState = "Click on it again!";
                    
    if (elementText.textContent === firstState) {
        elementText.textContent = secondState;
        elementText.style.backgroundColor = "#7ce5ff";  //adds background color
        elementText.style.padding = "3px"; //adds padding
        elementText.style.borderRadius = "10px"; //adds border radius
        elementText.style.textAlign = "center"; //adds text alignment
        elementText.style.color = "#ffffff"; //adds font color
    } else {
        elementText.textContent = firstState;
        elementText.style.backgroundColor = ""; //removes background color
        elementText.style.padding = ""; //removes padding
        elementText.style.borderRadius = ""; //removes border radius
        elementText.style.textAlign = ""; //removes text alignment
        elementText.style.color = ""; //removes font color
    }
}

var btn1 = document.getElementById("button1");
btn1.addEventListener("click", changeText, false);

Now we know how to change the appearance of an element, but could we make it entirely disappear by using the same style property? Actually, we can.

By accessing CSS properties through the DOM, even the visibility of elements can be controlled so that they either appear or disappear on the web page. This is extremely useful for, say, revealing and hiding menus.

If you want to be able to toggle visibility of our heading, adding a secondary button, a secondary event listener, and a secondary function (with similar if and else statements as before) to change the CSS display property values will work. Here is the revised code to accommodate the ability to toggle visibility on and off:

HTML Code Snippet:

                        
<h2 id="changingH2">Clicking on buttons is fun!</h2>
<button type="button" id="button1">Toggle Style</button>
<button type="button" id="button2">Toggle Visibility</button>

Notice that, to avoid confusion, the button names were changed to clarify what actions they trigger.

JS Code Snippet:

                        
function changeText() {
    var elementText = document.getElementById("changingH2");
    var firstState = "Clicking on buttons is fun!";
    var secondState = "Click on it again!";
                    
    if (elementText.textContent === firstState) {
        elementText.textContent = secondState;
        elementText.style.backgroundColor = "#7ce5ff";
        elementText.style.padding = "3px";
        elementText.style.borderRadius = "10px";
        elementText.style.textAlign = "center";
        elementText.style.color = "#ffffff";
    } else {
        elementText.textContent = firstState;
        elementText.style.backgroundColor = "";
        elementText.style.padding = "";
        elementText.style.borderRadius = "";
        elementText.style.textAlign = "";
        elementText.style.color = "";
    }
}

var btn1 = document.getElementById("button1");
btn1.addEventListener("click", changeText, false);

/* new function and event handler below */

function changeVis() {
    var displayVal = document.getElementById("changingH2"); //accesses header

    if (displayVal.style.display === "none") { //if display is none,
        displayVal.style.display = "block"; //make display block
    } else { //else (if display is one of the visible values),
        displayVal.style.display = "none"; //make display none
    }
}

var btn2 = document.getElementById("button2"); //accesses visibility button
btn2.addEventListener("click", changeVis, false); //function triggers on click

Adding and Removing CSS Classes

Although we've now added and removed styling to the heading as we've wanted to, typing out multiple class properties inside if and else statements seems a bit tedious after a while. This example didn't include that many CSS properties, but it easily could have included a lot more. What if there was a way to reduce the amount of code that needed to be written, and to better separate our CSS styling from our script file?

One of the ways that CSS styling through JavaScript can be simplified is by using the className property. By using this property, you can set a CSS class to an element. While you can apply multiple classes to an element in one statement by separating the class names with spaces, in this example we will only create one class which we will add onto and remove from the text element when clicking the "Toggle Style" button. In this revised example, we will use the same HTML code snippet as the last example but move the CSS properties to the style sheet and apply the class name of those properties to the header element in JavaScript:

CSS Code Snippet:

                        
.headingClass {
    background-color: #7ce5ff;
    padding: 3px 3px 3px 97px;
    border-radius: 15px;
    color: #ffffff;
}

JS Code Snippet:

                        
function changeText() {
    var elementText = document.getElementById("changingH2");
    var firstState = "Clicking on buttons is fun!";
    var secondState = "Click on it again!";
                    
    if (elementText.textContent === firstState) {
        elementText.textContent = secondState;
        elementText.className = "headingClass"; //adds entire class
    } else {
        elementText.textContent = firstState;
        elementText.className -= "headingClass"; //removes entire class
    }
}

var btn1 = document.getElementById("button1");
btn1.addEventListener("click", changeText, false);

function changeVis() {
    var displayVal = document.getElementById("changingH2");

    if (displayVal.style.display === "none") {
        displayVal.style.display = "block";
    } else {
        displayVal.style.display = "none";
    }
}

var btn2 = document.getElementById("button2");
btn2.addEventListener("click", changeVis, false);

Clicking on buttons is fun!

The results rendered on the web page are exactly the same, but the code is much neater. As you can see, there's definitely more than one way to write a function that results in the same actions. In fact, there are even simpler and shorter ways to recreate this interactive example using JavaScript or jQuery and the toggle() method, but that's beyond the scope of this tutorial.

However, taking shortcuts can also make problems appear. In the above code, the text-align property did not properly display centered text, so the text had to be centered by changing the padding to move the text over to the right. Always consider how much troubleshooting you want to do or adjustments you want to make with your code when experimenting with shortening it.

What You Should Learn Next

Before moving on to another topic, practice various ways of accessing elements using different JavaScript methods, testing out different ways of changing their content and CSS properties. Depending on the situation, find out if it's easier to change individual properties in JavaScript, or if it's easier to just create a class in your style sheet and then add the class to your element through JavaScript. Once you feel comfortable doing that, begin learning how to create elements in JavaScript, how to add them to the DOM tree, and then practice styling those new elements with what you have learned in this tutorial. Afterwards, follow any other JavaScript tutorial to level up your skills, such as from W3Schools or other useful resources. Good luck!
